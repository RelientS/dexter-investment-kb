# AI大模型与Agent生态

> OpenAI、Anthropic、智谱等大模型竞争格局，Agent/GenUI/端侧/机器人等应用生态演进

## 核心论点

- OpenAI 8520亿估值含C+B端，OpenAI=Netscape是真AI应用龙头
- Web4=Agentic Web，agent取代手机app launcher
- 在通用agent里集成软件vs在软件里集成agent，后者意味着收缩/关停/吞并
- OpenAI路线：Codex app发展成桌面唯一超级app，吞并Atlas+ChatGPT
- 腾讯萎靡：想把Agent集成到已有软件而非反过来
- 字节豆包风险：纯问答无资产沉淀，替换成本为0
- PC复兴：桌面agent让中国完成之前中断的PC普及
- Anthropic短视取巧vs OpenAI押100%身家在下一代浏览器+Google
- 机器人/IoT是AI算力天梯倒数第二，基本面跟上估值时点靠后
- 本地软件agent是AI算力天梯终端，最远
- coding是agent必经之路，跟AGI没关系
- AGI必须圆形，不能局部脑残
- 统一通用Runtime+生态，Bun这种App Runtime减少context切换
- Qwen不能偏科vs GLM/MiniMax可走Claude偏科路线
- MCP本质是Web生态，国内空白
- 千问接阿里全家桶（高德/闪购/支付宝）很危险
- 在已有抽象层之上搭建新抽象层，就是套壳
- 存储是鱼尾，光不是鱼尾
- 需求本来就是技术给的，技术同样可以把需求拿走
- TSLA Optimus是AI算力天梯中机器人/IoT层的代表
- OpenAI不亏了投资者反而要撤，说明现金流回正慢的生意回报反而大
- SemiAnalysis去年收入2000万刀今年预计1亿，订阅40万substack第一，员工85人，创始人29岁
- 互联网泡沫教训：思科问题不是炒作估值而是电信资本开支断崖下降；当年互联网基建供给侧驱动，现在AI基建消费侧驱动算力始终不够用
- 智谱和minimax很纯但有其他风险，不是真的中国版Anthropic和OpenAI
- 推算OpenAI明年初上市就一万多亿市值1万亿太基础；VCX重仓Anthropic但溢价非常高
- 对散户来说OpenAI机会比Anthropic大得多——C端上限高但短期不兑现，B端Anthropic共识强一致但含水分；上市早争议大波动大才有捕获价值空间
- AI时代消费力重点不在于人们收入而在于很多传统消费需求不存在了；物质消费天花板很低只有精神消费含服务消费能无上限扩展
- 豆包最大难点是怎么让大众用户对AI有需求，目前靠病毒传播推token消耗不可持续；不要小看套壳和产品包装，没包装前的coding agent需求天花板更低
- 一个行业中物理世界的东西越少就越容易被AI加速；生物技术发展很慢AI加快不了依赖生物/化学/物理实验时间的部分
- 一直把智谱定位为中国Anthropic最早推荐时间有据可查；能跟OpenAI比的是字节；Anthropic估值利好即将港股上市的MiniMax与智谱
- C端OpenAI有可能卷赢Google，但MiniMax怎么卷赢字节？字节本来就是来卷别人的
- 京东做AI应主要做Tool而不是做agent搞错了重点；Agent与Tool的区别就像搜索引擎和网页的区别
- Anthropic的数据接下来会有大波动可能牵连美股AI大跌；现在的数据是有泡沫的，对这种波动满仓带杠杆硬扛
- 豆包幻觉多是产品设计问题，默认快速模式导致
- JS、PY、GO、RS之间本来就有亲缘关系
- Anthropic提交S-1申请IPO，blog都加载不出来了
- Google到处借钱卖股票融资建算力、Anthropic到处找算力，反证OpenAI没有浪费和冒进
- 腾讯小程序Agent方案局限旧生态，趋势是把GUI嵌在Tool里而非反过来，是典型创新者的窘境
- 没有ETH就不会有现在加密技术最大的出路——RWA（含稳定币）
- 其他头部人形机器人startup在做的叠衣服类方向需要更多皮层参与，比翻跟头类更难
- 点云（3DSG）是现在最火的技术
- 中国软件之前没做到全球化，一是语言文化劣势，二是发展阶段没到、跟随美国
- 字节减少第三方token额度也有鼓励用seed code吃自己狗粮的原因
- 字节一贯赚下沉市场的钱，要赚下沉市场的钱要从刚需入手——办公时的PPT和表格
- Coding agent的瓶颈是遇到bug修不好、不懂业务、调研能力不行、不能做高层任务
- 看好OpenAI，不像Google和Apple有历史包袱；拖Apple后腿的是现有App和开发者生态
- 直接接触用户的软件抽象层就是前端，最根本的是最high-level的抽象层
- 通用agent是平台/runtime和生态（相当于浏览器），很多垂直agent是伪需求
- Agent用的Tool分服务器端和客户端，目前起飞的是客户端Tool
- 全套客户端agent能力做得好就是Agentic OS，做得差就是山寨ChatGPT+SaaS式AI集成
- Apple WWDC比预想好、体验比Pixel上的gemini更易普及，但没突破性亮点
- 文字多、像chatbot的UI只是GenUI发展完善前的过渡状态
- 眼镜替代/结合手机侧重移动/户外，Apple最适合做眼镜，手机的宿命是给眼镜当算力盒子
- XR全面拥抱现实世界已在走出低谷，加密技术要走出低谷也只能靠这个
- OpenAI代码提交大幅增加但CICD遇到瓶颈，想摆脱GitHub
- 把内容当作HTML、AI当作浏览器引擎，让AI给你渲染
- 太空算力工厂是大饼，地面算力工厂是短期可兑现的成长空间
- seedance跑通了是字节AI 2026年的关键
- 现在不但代码没人看，论文也开始没人看了
- Anthropic数据接下来会有大波动可能牵连美股AI大跌
- Siri AI的运行完全没用到Gemini，用的都是AFM 3
- 现在AI行业并不在乎「便宜」这个点
- Claude Fable 5改按token付费后企业用不起账算不过来，不用产品化手段把token消耗包装成大众能用的东西结果会很糟
- 隐私和延迟在很多场景无法忽略，越涉及多模态交互和日常生活场景越无法回避，这些只能在端侧做
- 云端AI需要跟端侧AI配合；微信里的Agentic小程序就是端侧多模态小模型处理截图和DOM操作充当云端agent和客户端Tool之间的翻译
- 研发用aime生成的调研文档和方案文档不如自己用GPT-5.5 Pro做调研出方案
- AI的根本问题不是消费能力而是生产出来的是什么——能不能像iPhone那样无中生有创造新需求新现金流；目前只是改变存量财富分配结构，很多时候只是让人在老场景偷懒
- 某模型不会涨价只会越来越便宜，跟Anthropic不是一个路线
- 后端做UI最大的不足是真的在「糊UI」——做的是像一层薄纸一样的UI
- Apple液态玻璃UI、visionOS眼手交互、notion/飞书多维表格多人实时协作、富途这样独立维护的跨平台应用，都是「一层薄纸」解决不了的前端需求
- 即使只看「薄纸」内的前端需求，也有内容结构语义、实际数据、UI状态、视图渲染效果四个层面的分离与抽象问题，每层都是一个垂直领域
- 追觅Eclix俞雷称即将推出一款没有APP的AI手机
- 分享《打下万亿江山的存储大佬，终究成不了首富》，指向存储行业财富创造的天花板问题
- 国内智驾+机器人物理AI最看好小鹏和Momenta，两家企业基因都是软件、技术方向靠谱；Momenta抢跑港股IPO能打
- 鄙视国内语境下的L4（依赖人工编程的智驾），宁要资本主义的L2不要社会主义的L4
- 群核科技有通过视觉信息理解空间环境的能力，空间智能业务贡献仍很小，打新便宜可押注其从房地产转型空间智能的正向不确定性
- 智谱溢价飞轮：溢价越高越有钱投算力/换ETH，进一步成为基础设施支撑更高溢价；智谱没现金流所以只能卖股票融资
- 核心方法论是抽象：抽象=约束=节约=确定；领域知识要建在一定高度的抽象上，别把底层复杂性泄露给agent
- GenUI-first Web：设备从安装软件走向生成软件，按需为非技术用户生产软件；但端到端日抛软件不建在新抽象架构上没价值
- agent是胶水打通流程，前提是提供足够抽象；coding agent最大场景不是专业程序员
- 机构开始靠AI发掘因子，这些因子人类无法解释只知有效；最赚钱的量化是自营（不卖），其次私募，公募成本很低
- 用GPT-5.5搞了港股状态机，45个提交；还买了tushare套餐做量化研究
- Claude tag不是AI外包，是身处大量core群、积累大量工作经验、掌握大量内网工具权限、异步自主行动的独立同事
- AI异步产出的结果太多来不及消化看不完；只看一手原文包括AI原始结果，不用AI总结和摘要
