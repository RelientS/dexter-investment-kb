# AI 产业革命与宏观格局

> AI 作为下一代通用技术革命的 S 曲线演进、中美战略博弈、算力军备竞赛、产业代际节奏、过热泡沫争议与可持续拐点判断。

## 核心论点

- 互联网泡沫和现在AI基建的核心区别是供给侧驱动vs消费侧驱动(算力永远不够)
- AI资本支出与阿波罗计划对比仅为量级概念，真正有参考意义的是与历史上铁路建设和互联网建设对比；投资转化为实物资产后能否回收要看现金流
- 当前AI行业正处于5%到30%渗透率的跨越阶段，而机器人行业还未达到5%的early adopter阶段
- 除AI外的出口不可持续，新能源、机器人、全产业链都是AI基建驱动的；没了AI中国出口只剩南方国家市场
- 中国新能源产能一直严重过剩，需要指望AI无限创造新需求来消化
- 裁员是 AI 公司的利好信号,DeepL25%、CF20%、Upwork25%、Atlassian10% 裁员后财报暴涨
- 反对在本地建数据中心的人大概率平时不怎么用AI,感受不到收益
- 行业中物理世界的东西越少越容易被AI加速,生物技术依赖实验时间AI加速不了
- AI上游和重资产是确定性强的「防风墙」，其生息确定性可以抵消宏观风险冲击
- 太空算力工厂是远期大饼，地面算力工厂才是短期可兑现的成长空间
- 建设地面算力工厂需要处理监管、能源、英伟达供货（老黄）等多方问题，马斯克在这些方面都有经验积累
- 谷歌和xAI的GPU租赁业务在抬升GCP的配置需求
- 云 GPU 是 AI 算力需求核心红利,包括 NVDA 加 NeoCloud
- 云 DRAM 和 HBM 估值已含 2027 年增长,下游变数大,已进入鱼尾阶段
- 云 NAND 即 SSD 是产品而非商品,可绕开 HBM 瓶颈,代表是 SanDisk 和闪迪
- 云 CPU 需求会因 agent swarm 加 Tool Use 指数级增长,AMD 正在建仓
- 机器人和 IoT 是 AI 算力天梯倒数第二层,基本面跟上估值的时点靠后
- 本地软件 Agent 是 AI 算力天梯最远的终端层
- HBM 没有差异化,只有能做和不能做的区别,技术壁垒非技术领先优势
- DRAM 是 AI 公敌,阻碍 scaling law,会被下游严厉惩罚
- 推理 PD 分离将成为趋势,prefill 用 GDDR7,decode 用 SRAM 替代 HBM
- GPU 是最难卷的环节,国产算力先指望推理侧突破
- GPU和光通信是AI scaling law的一部分,HBM只与特定架构有关卡脖子状态不可持续
- 光通信在AI基建中用量持续增长,水平扩展和垂直扩展都要增加光的使用
- 硅光、光IO、异构计算、SRAM、TPU、英伟达推理方案都在减少对HBM的依赖
- 光通信新技术更适合半导体企业做,传统光模块厂面临的是赛道切换而非门槛竞争
- 英伟达核心是加速更底层通用计算,用CUDA生态找重复性再硬件并行
- GPGPU(SIMT)并发海量动态线程软硬结合无上限,ASIC像建好的产线不灵活
- CUDA的真正优势是能轻易获得性能优化,ASIC每次优化都要重新建产线
- 华为/AMD/TPU标称性能需使用方自己做工程优化,只有Anthropic和字节这类大厂玩得起
- 先进封装是多芯片之间系统集成的逻辑,不同于前道工艺和传统封装
- Apple/高通端侧芯片、HBM、光、国产算力都需要先进封装,信先进封装肯定没错
- NVL从单机柜扩展到多机柜后铜才有瓶颈,CPO和垂直扩展是两条路
- 英伟达CPX(同构大context)和LPX(GPU+LPU异构速度)是两条不同的Rubin路径
- 要说光连接/先进光连接/硅光,不要说光模块(传统器件),技术带来健康成长
- Computex上Marvell CEO和黄仁勋共同指出，连接已成为AI发展的新瓶颈
- 机架内单通道达到400G后铜缆触及距离极限，互联将全面转向光学方案
- 数据中心互联重心已从传统的南北向（用户与数据中心之间）全面转向东西向（数据中心之间的scale-across）
- 英伟达NVLink路线演进：NVL72（单机柜72 GPU，铜NVLink）→ NVL576（8机柜576 GPU，铜光混合NVLink）
- NVL576要等柜内GPU密度和托盘间带宽增加后才会把铜换成光，基于Rubin Ultra，2027年才会推出
- NVL576带来的新增光需求无法用传统可插拔光模块满足，对缺少硅光能力的厂商是实质性利空
- ASIC比GPU更依赖跨机柜大规模互联，因此对光互联的需求发展更快
- AI产业分层框架需修正：云这一层应改为neocloud，存储不止是HBM，终端应用不止PC和手机，电力还要包括merchant power相关业务
- HBM容量受限于芯片周长（「海岸线」），芯片边长决定了能放置多少HBM颗粒
- 突破HBM海岸线限制的5大方向：HBM垂直升级、2.5D封装人工岛、群岛拆分（AMD/ASIC路线）、三维垂直（HBM5但散热困难）、远海高速桥（英伟达+Marvell光互联），纯SRAM方案不在此列
- 光互联的终极形态是把光直接做进芯片内部实现片上光通信，最极致是实现纯光计算
- 算力芯片厂商必然会自建neocloud或扶持代理人做云，就像AI模型厂商必然会做应用和agent一样，是产业垂直整合的必然趋势
- Coding agent真正的瓶颈不是写文档对齐（可通过多写文档加约束解决），而是bug修不好越改越坏只能回滚、缺乏业务理解、调研能力不足、无法承担高层级任务
- 在垂直AI场景可以通过多种方案减少HBM依赖：如Groq/Cerebras完全不用HBM只用SRAM、用GDDR7放大上下文、光互联和先进封装让NAND分担、英特尔某方案用LPDDR5
- AI算力没有高低峰之分，全天都用满，因为推理需求小时会被训练需求占满
- 只支持推理的算力也能用于辅助训练等内部建设工作
- 「网络」一词不够本质，应该叫「连接」；scale-up涉及的连接深入基板、封装内部，不能再算网络。
- 冯诺依曼体系下存储和电力是唯二原罪，核心问题都是供应不足、成本太高；相比之下NAND的原罪少一些。
- 英伟达边缘算力产品一直有在销售
- 边缘算力受带宽限制,适合跑本地多模态
- Cerebras、LPU/Groq等是业界摆脱HBM瓶颈的努力,高速推理越大HBM可持续性越低
- 摆脱HBM的两条技术路径:LPU/Groq靠软件编译,Cerebras靠硬件大芯片
- GPU服务器软件层清零大家同一起跑线,公有云不掌握agent和全栈软件层会被neocloud冲击
- Cerebras相当于奥特曼个人投资的neocloud,新模型专门为其硬件设计
- 不是所有AI需求都需要长上下文,Cerebras侧重速度优先的场景
- 超大规模云厂商（Hyperscalers）的流动性不会断，有大量资金愿意借给它们，现金流稳健，还有Google与黑石合作建TPU neocloud这类加杠杆的省钱模式
- Cerebras的WSE4到2027年流片算进度快的，谷歌TPU v8也要到2027年才能放量，中间这段时间需要靠neocloud进展和技术突破支撑叙事
- Cerebras单芯片售价约300万，自带44G内存和90万核，通过在芯片内直接放SRAM避免使用HBM
- 晶圆面积有限导致增加SRAM必须减少计算核，Cerebras下一步的黑科技是把两块晶圆3D堆叠上下连接突破限制
- 储能也是AI电力的受益方向，xAI数据中心就使用了特斯拉储能
- AI电力受益于两头：把电力搬到数据中心旁边（模块化）、把数据中心建在闲置电厂/商用发电商旁边（colocation）
- 可以把充满专业术语的内容当作HTML，把AI当作浏览器引擎，让AI来「渲染」解释给你看
- H200 是中国 AI 唯一瓶颈,国产连 HBM3 都还在攻关
- 华为时间缩放定律本质是用封装+软硬创新代替几何缩微,先进封装是国产最强环节
- 所谓tao定律只是摩尔定律子集,但对先进封装和光的需求判断是靠谱的
- Cerebras靠前道工艺做超大芯片,华为靠芯片间通信+软件软硬架构创新
- 字节自研的类Groq专用芯片有望同时实现推理高速和深度思考能力兼得
- 字节自研芯片的主要目的是扩大供应、增加供应选择、应对供应瓶颈、保障自身需求，而非完全替代英伟达
- 在推理侧成本和性能优化上，字节所有方案都在尝试，英伟达整体性价比反而更好，但核心问题是能买到多少
- SemiAnalysis是专业度很高的半导体分析机构,创始人29岁员工85人
- 腾讯和支付宝把AI当流量入口做赚不到钱，这种AI流量都是「工业废水」
- Web4 就是 Agentic Web,agent 将取代手机 app launcher 成为下一代互联网入口
- 在通用 agent 里集成软件才是下一代范式,在软件里集成 agent 意味着收缩、关停或吞并
- OpenAI 的路线是 Codex app 发展成桌面唯一超级 app,吞并 Atlas 加 ChatGPT
- 腾讯萎靡的原因是想把 Agent 集成到已有软件而非反过来,方向错误
- 字节豆包有风险,纯问答模式无资产沉淀,用户替换成本为 0
- 桌面 agent 会让中国完成之前中断的 PC 普及,PC 将迎来复兴
- 统一通用 Runtime 加生态是方向,Bun 这类 App Runtime 能减少 context 切换
- Cloudflare是agent infra和Tool infra,不是算力infra,NET已持有多年
- 只要价值来源是减少摩擦,不管是软件/SaaS还是真人岗位都会因AI被消灭
- Agent开发被前端夺舍了
- OpenAI收购Astral(uv/ruff/ty)做Python工具链,站错队了
- AI产品免费模式的核心目的是烧出新分发入口和用户习惯，如果因幻觉问题连「取代搜索引擎」这个最基本场景都没烧出来，投入就打水漂了
- AI调研工作流中一般不需要注明搜索范围，只有特定任务才需要明确范围且可通过封装Skill实现
- 腾讯小程序Agent方案类似opencli操作web，缺乏明确的Tool定义，只能从HTML/UI结构视觉操作并提取服务端API，实现效果难且token消耗大
- 腾讯可以通过离线索引所有小程序源码、提取API打WebMCP标记或自动写Skill的方式，来节省线上token消耗
- 前端的本质定义是「直接接触用户的最高层级软件抽象层」，当用户从人类变成agent后，前端就变成了做agent相关的工作；前端的根本不是用户，而是最高层级的抽象层，即使没有真人用户它也必然存在
- 通用agent本质是一个平台/runtime和生态，地位相当于浏览器时代的浏览器
- 很多垂直agent和集成在现有软件里的agent是伪需求，只有通用agent的需求是刚性且注定少不了的
- 当前通用agent特别依赖CLI形态的Tool，而很多CLI工具就是前端工程师开发的
- Agent用的Tool分服务器端（MCP、Skill封装的Open API）和客户端（CLI、WebMCP/小程序MCP、App Intents等），harness框架也对应分两端
- 当前AI Tool生态中发展最快、最先起飞的是客户端Tool而非服务端Tool
- 全套客户端agent能力做得好就是Agentic OS，做得差就是山寨ChatGPT加SaaS式AI能力集成没有本质差异
- Shortcuts在WWDC中变成了vibe coding app和自定义Tool生成器（生成Tool而非新GUI应用）
- 当前文字多、像chatbot的UI只是GenUI发展成熟前的过渡状态；建立下一代交互/UI范式的行业标准只能指望Apple
- 眼镜和目镜有明确分工：目镜替代/结合笔记本，主打室内生产力和沉浸式重度娱乐；眼镜替代/结合手机，主打移动/户外场景
- 智能眼镜的交互是当前最大难题（传感器数量不如目镜多，多模态自然交互体验差）
- 云端AI必须跟端侧AI配合，类似人形机器人架构；微信Agentic小程序就是端侧多模态小模型充当云端agent与客户端Tool的翻译

- 已从modern web转向spatial web和agentic web（tool组成的web、agent自己玩的web），参考重仓的Cloudflare。
- GenUI-first Web（设备从「安装软件」走向「生成软件」）规模庞大有乘数效应，但端到端「日抛软件」若不建在新抽象架构上没价值，本质是tokenmaxxing。
- 看好Google A2UI+MCP Apps和追觅「没有APP的AI手机」这类带抽象层的GenUI方向。
- agent是「胶水」用于打通流程，前提是提供足够的抽象层；coding是顶级专业人类使用工具的最高效手段，agent应该像「最顶级专业的人类」那样跟世界打交道。
- coding agent的最大场景不是专业程序员；物理世界AI铺开需要「宇树」（肢体/神经/小脑）+「momenta」（数据飞轮训练的感官和运动皮层）。

- Qwen(千问)不能偏科,而 GLM 和 MiniMax 可以走 Claude 的偏科路线
